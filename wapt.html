Web Application Penetration Testing

Web Application Penetration testing is majorly used to identify vulnerabilities in your web applications and advise you on how to fix them, before hackers exploit them.

We perform Web App Penetration Tests from a blackbox perspective and cover the OWASP Top 10 vulnerabilities.

Below are steps followed in WAPT:

  • Lockheed Martin Cyber Kill Chain- We will focus on the Adversary side of the Kill Chain
  • Reconnaissance - gaining information on the target - harvest email addresses, IP addresses, host/network information, vulnerability identification, identify employees on social media, press releases, contracts awarded, discover Internet-facing servers.
  • Weaponization- attackers obtain a “weaponizer” (tool that couples malware and an exploit into a deliverable payload) from public/private channels or build in-house.
  • Delivery- adversaries have launched the malware to the target. Adversary Controlled Delivery (direct against web servers).
  • Lockheed Martin Cyber Kill Chain- We will focus on the Adversary side of the Kill Chain
  • Exploitation- Attackers must exploit a vulnerability to gain access/Zero-Day exploits. Software, hardware, or human vulnerability.
  • Installation- Attacker wants to maintain access, so they typically install a backdoor at this stage.
  • Command & Control (C2)- The malware opens a channel of communication, so the attacker can manipulate the victim remotely.
  • Actions on Objectives- Attackers now have “hands on keyboard” and move forward with their objective.
